MySQL内部模块（持续更新中...）

发表于 2021-06-27 | 分类于 数据库 ， MySQL | | | 次阅读

前言

这篇文章演示环境为：MySQL5.7。

主要内容为基于MySQL5.7官方文档的学习，对MySQL整体架构及内部模块相关内容的整理及自己的理解。

如有错误或疑问，欢迎讨论！

整体架构

物理架构

Base Directory

MySQL二进制文件目录

Program log files(程序日志文件)：

主要为MySQL进程的PID、Socket、文档、依赖等文件。

Program executable files(程序可执行文件)：

主要为MySQL启动、备份等功能的可执行文件。

Data Directory

MySQL的数据、状态、日志文件目录

Data Directory(数据文件)：

主要为MySQL服务端日志、状态日志、存储引擎、表空间等数据文件。

Data sub-directory(数据子文件)：

主要为MySQL保存或生成的数据、索引、结构文件。

逻辑架构

Client（客户端）

用于连接MySQL服务器的各种应用程序

Server（服务端）

MySQL实例，用于进行数据处理及数据存储

Connections（连接器）

主要负责管理客户端连接/会话。

MySQL连接方式主要分为两种：

交互式连接：

在mysql_real_connect()中使用CLIENT_INTERACTIVE选项的客户端产生的连接就是交互式连接，例如通过命令登录MySQL进行操作，当你停止操作时间超过参数interactive_timeout后，MySQL就会断开连接；

非交互式连接：

在mysql_real_connect()中使用wait_timeout选项的客户端产生的连接就是非交互式连接，例如通过JDBC等方式产生的连接，当非交互式连接获取的线程不活跃时长草果wait_timeout（默认8小时）后，MySQL就会清除掉该连接线程。

Query Cache（查询缓存）

用于存储SELECT语句的文本以及发送给客户端的相应结果。

Query Cache(查询缓存)在 MySQL 5.7.20 时已弃用，并在 MySQL 8.0 中删除。

Parser（解析器）

用于检查SQL语句中的每个字符，来解析语义、词义、语法，并为每个SQL生成SQL_ID。

用户凭据检验是在解析器中检查的。

Optimizer（优化器）

优化SQL语句并根据存储引擎生成成本最低的执行计划。

用户访问权限实在优化器中检查的；

成本最低 ≠ 时间最短

Storage Engine（存储引擎）

存储引擎，主要负责管理物理数据、执行SQL语句并从数据文件中获取数据

MySQL可支持存储引擎

常见存储引擎：

  • InnoDB（MySQL5.5.5版本之后的默认存储引擎）
  • MyISAM
  • MEMORY
